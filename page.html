Name: Ethical Hacking - The OWASP Juice Shop Challenge
Address: 798 Washington Street
Time: 02:45
Description

Help us hack a fake website! As web applications become increasingly common, keeping them secure is more important than ever. In order to defend against such attacks, you have to understand how they work, and what better way to learn than to see them in action? Join us as we attack the OWASP Juice Shop.

Overview:

After going over the scenario and concepts involved at a high level, we will create a running instance of this app and see how many vulnerabilities we can identify and exploit. We'll discuss the weaknesses we find and how to protect against them.

About the Event:

You don't need to be a techie to get something out of this event! We will make the initial discussion of the topic accessible to anyone curious enough to stop by. As for tools, you really only need something with a web browser and a physical keyboard. It would be useful (but not required) to know a bit about dev tools. Here is a free interactive way to learn about them:

http://discover-devtools.codeschool.com/


For more advanced users, we recommend having Kali Linux or some other virtual machine installed on your system. As an alternative, you could use the free version of the Burp proxy from portswigger.net or any other similar proxy tool.

Please do not look up any solutions to the challenge ahead of time (that ruins the fun!)

Background:


In 2014, a group of volunteers built a highly insecure web app designed to support a fictional juice shop. They created the app as a tool for demonstrating important security vulnerabilities existing in actual websites.

In 2016, the Open Web Application Security Project (OWASP) began sponsoring the Juice Shop demo. OWASP is a nonprofit organization that maintains a list of the ten most important security flaws for web applications. The Juice Shop project illustrates the exploits that the organization wants to raise awareness about.
